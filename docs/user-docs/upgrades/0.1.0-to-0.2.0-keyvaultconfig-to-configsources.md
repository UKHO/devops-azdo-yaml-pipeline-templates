# Upgrade Guide: 0.1.0 → 0.2.0 - KeyVaultConfig → ConfigSources

This guide shows how to move toward the preferred `ConfigSources` array format while keeping `KeyVaultConfig` supported for existing setups.

## Upgrade Path

If you currently use a single Key Vault with `KeyVaultConfig`, you can keep using it. For new work or updates, prefer `ConfigSources` and convert the object to a one-item array when you are ready.

## Coexistence Rule

`KeyVaultConfig` and `ConfigSources` cannot exist at the same time in the same `TerraformDeploymentConfig`. Choose one approach per configuration.

```yaml
TerraformDeploymentConfig:
  AzDOEnvironmentName: 'prod-environment'
  RunMode: 'ApplyOnly'
  ConfigSources:
    - Type: 'KeyVault'
      Name: 'prod-secrets-vault'
      ServiceConnection: 'Azure-Prod-SC'
      SecretsFilter: 'terraform-*'
```

## What Changed

- `ConfigSources` is the preferred option going forward
- Move the existing values into the first array item
- `Type`, `Name`, `ServiceConnection` remain required
- `SecretsFilter` defaults to `'*'`
- `RunAsPreJob` remains optional and defaults to `false`
